| 상태 | 기능 정의 완료 |

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

기존 알람·메모·캘린더 앱들은 각각 분리되어 있거나, 알람 그룹 조건부 ON/OFF, 메모 텍스트 기반 자동 알람 생성 등 핵심 기능이 없음. 본 앱은 이 세 기능을 하나의 앱에서 유기적으로 연동하는 것을 목표로 함.

### 1.2 핵심 가치

- 메모를 쓰면 자동으로 알람이 만들어진다
- 알람을 그룹으로 묶어 조건부로 켜고 끈다
- AI가 급하게 쓴 메모를 정리해준다
- 웹에서 시작해 Electron → 모바일 앱으로 확장한다

## 2. 기능 상세 정의

### 2.1 알람 (Alarm)

#### 2.1.1 알람 그룹 관리 (CRUD)

- 그룹 생성/수정/삭제
- 기타 그룹은 삭제 불가 (기본값)
- 그룹별 색상, 아이콘 설정

#### 2.1.2 알람 그룹 토글

- 각 그룹 옆에 ON/OFF 토글 버튼 상시 노출
- 그룹 OFF 시 해당 그룹의 모든 알람 비활성화
- OFF 상태 그룹은 dimmed 처리 (opacity 0.35)
- 그룹 ON 복구 시 개별 알람 기존 상태 유지

#### 2.1.3 알람 설정

- 마지막 사용한 소리/진동 설정을 디폴트로 유지
- 소리, 진동, 반복 요일, 라벨, 그룹 지정 가능
- 스누즈 지원

#### 2.1.4 빠른 알람 (Quick Alarm)

- 자연어 입력: "30분 후", "내일 9시 기상"
- 파싱 결과 확인 후 그룹 선택 → 저장
- 파싱 실패 시 안내 메시지 표시

#### 2.1.5 휴지통

- 삭제 시 30일간 휴지통 보관
- 30일 후 자동 영구 삭제
- 복원 가능

---

### 2.2 메모 (Memo)

#### 2.2.1 기본 메모 기능

- 제목 + 본문 자유 형식
- 최신순 목록
- 작성 시각 자동 기록

#### 2.2.2 위치 자동 태깅

- 메모 저장 시 GPS 좌표 자동 수집
- 역주소 변환 후 메모에 표시 (예: 강남역 근처)
- 위치 권한 거부 시 null 저장 후 정상 진행

#### 2.2.3 AI 메모 정리 (핵심 기능)

급하게 작성된 메모를 Llama가 분석하여 구조화된 내용으로 재정리.

| 항목 | 설명 |
|---|---|
| 원문 보기 | 사용자가 작성한 원본 메모 그대로 표시 |
| AI 정리 보기 | AI가 핵심 내용 파악 후 읽기 쉽게 재구성 |
| 토글 버튼 | 원문 ↔ AI 정리 언제든 전환 가능 |
| AI 처리 시점 | 메모 저장 시 백그라운드 자동 처리 |
| 수동 재분석 | 버튼으로 AI 재분석 요청 가능 |

#### 2.2.4 자동 알람 감지 (메모 연동)

- 메모 저장 시 Llama가 날짜/시간 패턴 자동 감지
- 예: "내일 9시 OOO" → 내일 오전 9시 알람 생성 제안
- 감지 후 사용자 confirm → 알람 생성
- 감지된 알람 제안은 메모 상단에 배지 형태로 표시

#### 2.2.5 휴지통

- 삭제 시 30일간 보관 후 자동 영구 삭제

---

### 2.3 홈 (Home)

- 자연어 입력창: 무엇이든 입력하면 Llama가 카테고리 분류
- 분류 결과: 일정 / 알람 / 할일 / 메모 / 나중에 / 언젠가
- 분류 결과 confirm 후 저장 (잘못된 분류 시 수동 변경 가능)
- 오늘 요약, 다음 알람, AI 제안 패널

---

### 2.4 나중에 알려줘 (Later)

- 지금 당장은 아니지만 잊기 싫은 항목 저장
- 알림 시간 선택: 오늘 저녁 / 이번 주말 / 다음주 / 직접 설정
- 알림 수신 후 완료 또는 미루기
- 휴지통 적용

---

### 2.5 언젠가 리스트 (Someday)

- 언젠가 하고 싶은 것들을 모아두는 버킷리스트
- Llama가 카테고리 자동 분류: 여행 / 배움 / 구매 / 기타
- 수동 카테고리 변경 가능
- 즐겨찾기
- 휴지통 적용

---

### 2.6 캘린더 (Calendar)

- 월간 캘린더 뷰
- 알람/메모가 연동된 날짜 dot 표시
- 날짜 탭 시 해당 날 일정 목록 표시

---

### 2.7 설정 (Settings)

- 시간 표시 형식: 24시간 / 12시간
- 테마: system / light / dark / purple / blue (5종, OS 다크모드 추종)
- 표시 언어: 한국어 / English (i18next)

## 3. 기술 스택

| 영역 | 기술 | 비고 |
|---|---|---|
| 프론트엔드 (웹) | React + TypeScript + Vite | PWA |
| 스타일 | Tailwind CSS | 웹/Electron |
| 모바일 스타일 | NativeWind v4 | Tailwind 호환 |
| 백엔드/DB | Firebase Firestore | 실시간 동기화 |
| 인증 | 카카오 로그인 + Firebase 커스텀 토큰 | 방법 A |
| AI | Llama | 호스팅 방식 미정 |
| 알람 (웹) | Web Audio API + Service Worker | 브라우저 푸시 |
| 알람 (Electron) | node-schedule + OS 네이티브 알림 | |
| 알람 (모바일) | FCM (Firebase Cloud Messaging) | |
| 기기 감지 | Firestore devices 컬렉션 실시간 업데이트 | |
| PC 설치형 | Electron | 웹 빌드 래핑 |
| 모바일 앱 | React Native + Expo | |
| 모노레포 | pnpm workspace + Turborepo | |
| 호스팅 | Firebase Hosting / Vercel | 미정 |

## 4. 개발 로드맵

| Phase | 목표 | 주요 기능 |
|---|---|---|
| Phase 1 | 웹 MVP | 알람 그룹/CRUD, 메모, 캘린더, 홈 기본 |
| Phase 2 | AI + 기기감지 | Llama 연동, FCM, 기기 라우팅, 나중에/언젠가 |

| MVP 이후 | 컨텍스트 알람 | PC/모바일 활동 감지 기반 조건부 알람 |

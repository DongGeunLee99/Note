# SmartNote QC 시나리오 v0.1

> Phase 1 웹 MVP 수동 검수용 시나리오. 코드 동작 기준으로 작성 (2026-06-10).
> 결과 기록: ✅ 통과 / ❌ 실패 / ⏭ 보류 — 실패 시 비고에 재현 절차·스크린샷 첨부.

---

## 테스트 환경

| 항목 | 내용 |
|---|---|
| 실행 | `pnpm dev` → `http://localhost:5173` (dev 서버 기준) |
| 브라우저 | Chrome 최신 (OS 알림 테스트는 브라우저 알림 권한 필요) |
| 데이터 | 로컬 목업 (`mocks/mockData.ts`) — **새로고침 시 초기화됨** (설정·패널 너비 제외) |
| 범위 외 | 프로덕션 빌드(`vite build` 기존 이슈), Firestore 연동, 실제 Llama AI, FCM |

### 공통 전제
- 모든 데이터 조작(추가/수정/삭제)은 새로고침하면 목업 초기값으로 돌아간다. **유일한 영속 항목: 시간 형식, 우측 패널 너비 (localStorage)**
- AI 분류·요약·알람 감지는 목업 로직(정규식 + chrono-node)이므로 아래 명시된 입력 예시를 그대로 사용할 것
